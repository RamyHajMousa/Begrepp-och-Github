Github flow

Det är viktigt till varje programmerare att lära känna GitHub Flow, särskilt om man vill jobba på grupper.

Vad är GitHub Flow?

GitHub Flow är ett enkelt arbetsflöde för skapa och hantera brancher när man jobbar på personliga projekt eller på stora projekt med andra utvecklare.

Hur använder man det?

GitHub Flow består av några steg börjar med att skapa ett utrymme för att skriva eget kod utan att påverka den huvud koden. Dessutom GitHb Flow ger dina medarbetare ett möjlighet att kolla och granska vår kod. Innan kan man använda GitHub Flow, måste man skapa ett konto på GitHub webbsida, installera Git på datorn och ha ett kod editor för att skriva kod på, tex VS Code. Sedan kan man hitta alla GitHub Flow steg på den här länk GitHub Flow.

Vilka viktiga delar är inkluderade i begreppet?

De viktiga delar i GitHub Flow är:

  1. Skapa en branch i ditt repo med ett bra namn som beskriver ditt arbete.
  2. Göra egna ändringar på repot som skapa, ta bort och ändra innehållet i filer.
  3. Commit dina ändringar med en beskrivande text som beskriver dina kod eller ändringar.
  4. Pusha ändringarna till repot på GitHub webbsida.
  5. Skapa Pull Request när du är klar för att dit arbete granskas. Pull Request skapas med ett kort beskrivande text om dit arbete.
  6. Merge din Pull Request med huvud kod när den blivit granskad.
  7. Ta bort din skapade branche när arbetet och alla steg ovan är klar.

Flexbox

Vad är Flex Box?

Flexbox är en CSS modul för att sortera och manipulera elementer på webbsidan. Den är mycket populär eftersom den är responsiv och flexibel särskilt på små skärmar.

Hur fungerar Flex Box?

Om man vill använda Flexbox för att sortera Children elemnter, måste man skriva property display: flex; på Parent elements CSS kod. Flexbox sorterar elementer vertically och/eller horizontally. Man kan bestämma vilken axel med property flex-direction som har fyra värde: row, column, row-reverse och column-reverse. Sedan kan man manipulera elemnter med andra properties tex: flex-wrap, flex-grow, flex-basis, align-items och justify-content ...

DOM (Document Object Model)

Vad är Dom?

Dom är ett programmeringsgränssnitt eller trä av "nodes" som ger JavaScript chans att ändra och manipulera alla elementer på HTML dokument och sina attribut och style.

Genom DOM kan JavaScript:

  • Skapa, ta bort och ändra elementer.
  • Skapa, ändra och ta bort elements attribut.
  • Skapa, ändra och ta bort CSS properties för att styla webbsidan.

Hur manipulerar man DOM?

Enlig ovanstående, kan vi manipulera DOM genom JavaScript. Vi kan välja vilken element som helst genom objektet "document" och några metoder som getElementById(); getElementsByClassName;() getElementsByTagName(); eller querySelector(); ... Sedan kan vi manipulera elementen med JavaScript metoder och properties.

Fin kod

Egenskaperna för finkod:

  • Läsbarhet: En mycket viktig princip när man skriver kod är att komma ihåg att kanske en annan programmerare kommer att jobba eller förbättra koden. Därför bör man försöka följa KISS-principen (Keep It Simple, Stupid!) och YAGNI-principen (You Aren't Gonna Need It). KISS-princip säger att de flesta system fungerar bäst om koden är enkel. Det bör följa DRY-regeln också (Don't Repeat Yourself) dvs använda metoder istället for skriva samma kod igen.
  • Underhållbarhet: Underhållbarhet mäter hur lätt koden kan underhållas. Det relaterar till kodens struktur, storlek och komplexitet.
  • Återanvändbarhet: Återanvändbarhet mäter om befintliga kod kan användas igen.
  • Testbarhet: Det mäter hur väl programvaran stöder testinsatser. Kod utan tester är inte ren kod.

Hur kan man uppnå god kodkvalitet?

  1. Analysera koden.
  2. Använd en kodningsstandard.
  3. Följ bästa praxis för kodgranskning.
  4. Refaktorera äldre kod (vid behov).
  5. Öva på regelbundet.

Ramverk

Inom webbutveckling använder man begreppet ramverk, vad är det? Ramverk är ett mjukvara för att hjälpa webbutvecklare att skapa webbsidor och applikationer. De kan spara tid och ansträngning särskilt på stora projekt och det används mycket på Webbutveckling.

Det finns många olika ramverk till Webbutveckling till exempel:

  • Backend ramverk:
    • Django
    • Express
    • Laravel
    • Rails
    • Spring
  • Frontend ramverk:
    • React
    • Angular
    • Vue

Jag ska prata lite om två frontend ramverk:

  1. React: i själva verket, React är inte ett ramverk, utan den är ett bibliotek. Den har utvecklats av Facebook. Många webbutvecklare använder React eftersom den är mycket snabbare och ganska lätt att manipulera DOM.
  2. Angular: Den utvecklades av Google och Microsoft använder den. Angular är bra att utveckla Single-page applikationer. Den är svårare att lära sig än React och storre än andra ramverk.
